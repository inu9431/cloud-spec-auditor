# CostCutter — AI가 클라우드 비용 전문가를 대신하는 시대

> **"코드를 만드는 건 AI가 해결했다. 이제 남은 문제는 — 그 코드를 세상에 내놓는 비용이다."**

---

## 이 프로젝트가 시작된 이유

저는 코딩 부트캠프를 수료한 뒤 같은 부트캠프에서 조교로 일하기 시작했습니다.

수강생 질문에 일일이 답변하는 업무가 반복되자, 자동화를 떠올렸습니다.
디스코드 봇을 만들어 **수강생 질문 → LLM 답변 → Django 검증 → 노션 업로드**까지 자동화했습니다.

그리고 이 봇을 AWS에 배포했을 때, **예상하지 못한 비용이 발생했습니다.**

프리티어를 쓰고 있었지만 EC2와 RDS 조합만으로도 비용이 생겼고,
어떤 인스턴스가 과스펙인지, 다른 클라우드가 더 싼지 판단할 기준이 없었습니다.

**인프라를 아는 개발자인 저도 이랬습니다.**
그렇다면 바이브코딩으로 처음 서비스를 배포하는 사람은 어떨까요?

---

## 문제: 만드는 건 쉬워졌지만, 운영하는 건 여전히 어렵다

AI가 코딩을 대신하는 시대가 왔습니다.

Cursor, Claude Code, Replit Agent 같은 도구 덕분에 이제 기획만 잘하면 누구나 앱을 만들 수 있습니다. 비개발자도 아이디어를 프로토타입으로 바꿀 수 있는 시대입니다.

**하지만 만든 앱을 세상에 내놓으려면 클라우드에 배포해야 합니다.**
그리고 그 순간부터 전혀 다른 문제가 시작됩니다.

- AWS, GCP, Azure — 어디가 더 싸지?
- 지금 쓰는 인스턴스가 과스펙인 건 아닌지?
- 각 클라우드마다 다른 도구, 다른 가격 체계, 다른 용어

이 질문에 답하려면 **인프라 전문 지식**이 필요합니다.
기존 비용 분석 도구(AWS Cost Explorer, Vantage, Infracost)는 이미 인프라를 아는 DevOps 엔지니어를 위해 만들어졌습니다.

그래서 지금까지의 해법은 **사람을 고용하는 것**이었습니다. 클라우드 아키텍트나 DevOps 엔지니어를 채용해 인프라를 관리하는 방식입니다. 하지만 인프라 전문가 한 명의 연봉은 최소 6,000만~1억 원 수준입니다. 10인 이하 스타트업이나 1인 개발자가 감당할 수 있는 비용이 아닙니다.

**결국 인프라 전문가가 없는 팀은 과스펙 인스턴스를 모른 채 비용을 낭비하거나, 감으로 인프라를 선택하고 있습니다.**

---

## 해결: AI가 클라우드 인프라 전문가 역할을 한다

CostCutter는 클라우드 비용 관리에 전문 지식이 없는 사람도 인프라를 최적화할 수 있게 해주는 AI 플랫폼입니다.

### 사용자가 하는 일

AWS Key를 한 번 등록합니다. **끝입니다.**

### AI가 하는 일

1. 현재 인프라를 자동으로 수집합니다
2. AWS, GCP, Azure 3사를 동일한 기준으로 비교합니다
3. 과스펙 인스턴스를 탐지합니다
4. 비용 절감 방법을 **사람이 이해할 수 있는 언어로** 설명합니다

### 예시 결과

```
현재 인프라:  AWS t3.medium  (월 $84)
  - 평균 CPU 사용률: 8%
  - 메모리 사용률: 22%

AI 추천:
  "이 인스턴스는 CPU와 메모리를 20% 미만으로 사용하고 있어 과스펙입니다.
   t3.small로 변경하면 월 $42를 절감할 수 있습니다.
   GCP e2-small을 사용하면 동일 스펙에서 월 $38로 더 절감됩니다."

예상 절감: 연간 약 $504
```

## 왜 기존 서비스와 다른가

| 구분 | 기존 방식 | CostCutter |
|---|---|---|
| **타겟 사용자** | DevOps 엔지니어, 인프라 전문가 | 1인 개발자, 초기 스타트업, 비개발자 메이커 |
| **인프라 전문가 비용** | 연 6,000만~1억 원 채용 필요 | AI가 대체 — 추가 인건비 없음 |
| **클라우드 범위** | 단일 클라우드 내 분석 | AWS / GCP / Azure 3사 통합 비교 |
| **사용 방식** | 대시보드에서 직접 분석 | Key 등록 한 번이면 자동 수집 + 자동 분석 |
| **결과 형태** | 숫자와 차트 | AI가 왜 이 추천을 했는지 자연어로 설명 |
| **비교 기준** | 각 클라우드 자체 단위 사용 | 3사 리전을 동일 기준으로 정규화 후 비교 |

### 핵심 차별점: 3사 리전 정규화

AWS는 `ap-northeast-2`, GCP는 `asia-northeast3`, Azure는 `koreacentral`이라고 부릅니다.
같은 서울 리전인데 이름이 전부 다릅니다.

CostCutter는 이런 리전명을 하나의 정규화된 기준(`NormalizedRegion.KR`)으로 통합합니다.
이를 통해 **사과 대 사과 비교**가 가능해집니다.

## 기술 구조: 왜 이렇게 만들었는가

### 데이터 파이프라인 (ELT 구조)

수집된 데이터는 3계층 파이프라인을 통해 처리됩니다.

**Bronze Layer — 원본 보존**

외부 API 응답을 가공 없이 즉시 저장합니다. Append-Only 방식으로 수정/삭제가 불가능하며, SHA-256 해시로 동일 데이터를 중복 저장하지 않습니다.

왜? AI 분석 프롬프트나 정규화 로직이 바뀌었을 때 과거 데이터에 소급 적용하려면 원본이 반드시 있어야 합니다. 초기에는 원본을 저장하지 않는 ETL 구조였지만, Backfill 불가능 문제를 발견하고 ELT로 전환했습니다.

**Silver Layer — 정제된 데이터**

원본 데이터를 읽어 서비스 규격에 맞게 변환합니다. 리전 정규화, 인스턴스 상태 표준화가 이 단계에서 이루어집니다. idempotent한 `update_or_create`로 동일 데이터를 여러 번 처리해도 결과가 같도록 보장합니다.

**Gold Layer — AI 분석 결과**

정제된 데이터를 바탕으로 AI가 비용 절감 방안을 분석합니다. Python이 수치를 계산하고, LLM은 설명 생성만 담당합니다. 이 분리 구조는 LLM의 수치 계산 오류(환각)를 구조적으로 방지하기 위한 설계입니다.

### AI 분석의 2단계 구조

CostCutter의 AI 추천 시스템은 의도적으로 두 단계로 분리되어 있습니다.

**1단계 — Python 분석 레이어:** 리소스 사용률 분석, 비용 계산, 최적화 후보 탐지. 모든 숫자 계산은 결정적(deterministic) 로직으로 수행합니다.

**2단계 — LLM 레이어:** 1단계의 분석 결과를 받아 사람이 이해할 수 있는 형태로 설명을 생성합니다.

왜 분리했는가? LLM에게 "이 인스턴스의 월 비용을 계산해줘"라고 하면 정확하지 않은 결과가 나올 수 있습니다. 숫자는 Python이, 설명은 AI가 담당하는 구조로 신뢰성을 확보했습니다.

### Prefect 오케스트레이션

데이터 수집과 처리는 Prefect로 자동화됩니다. `@flow`와 `@task` 데코레이터로 파이프라인을 구성하고, 24시간 주기로 자동 수집이 이루어집니다.

1인 개발 환경에서 Airflow는 별도 메타DB, 웹서버, 스케줄러 등 인프라가 과도했습니다. Prefect는 `prefect server start` 한 줄로 UI까지 실행 가능해 운영 단순성을 선택했습니다.

## 운영 안정성: 실제 배포를 고려한 설계

### 보안

- AWS Key 등 민감 정보는 **Fernet 대칭 암호화**로 DB에 저장 (평문 저장 방지)
- 복호화는 AWS API 호출 시점에만 수행하여 노출 최소화
- CostCutter는 **읽기 전용 권한만 사용** — 인스턴스 중지/삭제 등 쓰기 작업 없음

### API 남용 방지

봇이 sync 엔드포인트를 반복 호출하면 **유저의 AWS 계정에서** Cost Explorer API 비용이 발생합니다. 서버 보호가 아니라 **고객 보호** 차원에서 DRF Throttle을 적용했습니다.

| 엔드포인트 | 제한 |
|---|---|
| sync (인벤토리 수집) | 유저당 1회/10분 |
| audit (AI 분석) | 유저당 1회/5분 |
| 가격 sync | 전역 1회/1시간 |

### 장애 대응

- **Partial Failure 허용:** 인스턴스 1개 저장 실패 시 나머지는 계속 처리
- **Graceful Degradation:** Cost Explorer 권한 없으면 비용 0.0 반환 + 파이프라인 계속 진행
- **자동 재시도:** 외부 API 호출에 retries + exponential backoff 적용

## 성장 기록: 이 프로젝트에서 배운 것

### 디스코드 봇에서 시작된 기술적 성장

디스코드 봇을 만들 당시에는 DTO와 Adapter 패턴을 몰랐습니다. 강사님에게 코드 리뷰를 요청한 뒤 이 패턴들을 알게 되었고, CostCutter에서는 처음부터 **DTO + Adapter 패턴으로 외부 API 의존성을 분리**하는 설계를 적용했습니다.

디스코드 봇에서는 LLM API 호출 비용과 유사 질문 중복 호출 문제도 있었습니다. 이를 **PostgreSQL pg_trgm 확장으로 유사도 검색**을 구현해 해결했습니다. 이 경험이 CostCutter의 캐시 전략 설계에 직접적인 영향을 주었습니다.

### 방향성 결정

이 프로젝트는 백엔드, 클라우드, 데이터 세 영역을 모두 다룹니다. 세 방향을 다 깊게 가기에는 1인 개발의 한계가 있었기에, **데이터 파이프라인을 핵심 축으로** 설정하고 나머지는 이를 지원하는 구조로 잡았습니다.

### 기존 시장과 다른 포지션

클라우드 비용 관리 시장에는 이미 Vantage, Infracost, CloudHealth 같은 서비스가 있습니다. 하지만 이들은 **이미 인프라를 잘 아는 DevOps 엔지니어**를 타겟으로 합니다. 대시보드가 복잡하고, 전문 용어를 알아야 분석 결과를 해석할 수 있습니다.

CostCutter가 노리는 시장은 다릅니다. 바이브코딩으로 앱을 만들어 처음 배포하는 메이커, 인프라 전문가가 없는 1~3인 스타트업, 클라우드를 써야 하지만 비교 분석할 시간도 지식도 없는 개발자 — 이 사람들에게는 "대시보드를 줄 테니 알아서 분석해라"가 아니라 **"AI가 분석해서 왜 이게 좋은지 설명해주는"** 접근이 필요합니다.

이 프로젝트를 설계하면서 모든 주요 기술 선택을 ADR(아키텍처 의사결정 기록) 11건, 트러블슈팅 기록 12건으로 체계적으로 남겼습니다.

## 시스템 아키텍처

```
Cloud APIs (AWS / GCP / Azure)
         ↓
  Inventory Collector (Prefect @task)
         ↓
  [Bronze] Raw Data Storage (Append-Only, JSONB)
         ↓
  [Silver] Transform & Normalize (리전 정규화, 스키마 통합)
         ↓
  [Gold] AI Optimization Engine (Python 계산 + LLM 설명)
         ↓
  Dashboard / REST API (Django + DRF)
```

### 전체 데이터 흐름

1. **유저가 AWS Key를 등록합니다** → Django REST API로 암호화 저장
2. **Prefect 스케줄러가 24시간 주기로 수집합니다** → EC2 인스턴스 목록, 비용, 라이트사이징 추천
3. **Raw 데이터를 원본 그대로 저장합니다** → Bronze Layer (Backfill 가능)
4. **정제하고 정규화합니다** → Silver Layer (3사 통합 비교 가능한 형태)
5. **AI가 분석하고 설명합니다** → Gold Layer (Python 계산 + LLM 자연어 설명)
6. **대시보드에서 조회합니다** → 이미 처리된 결과를 서빙하므로 빠른 응답

## 향후 발전 방향

- **예측형 추천:** 시계열 Raw 데이터 기반으로 트래픽 트렌드를 분석하여, 3개월 후 인프라 부족/과잉을 선제적으로 알려주는 기능
- **비용 이상 탐지:** 평소 패턴 대비 급격한 비용 증가 시 자동 알림
- **FinOps 대시보드:** 멀티 클라우드 비용을 한 화면에서 추적하는 개발자 친화적 대시보드
- **Reserved/Spot 가격 비교:** 현재 On-Demand 기준에서 확장하여 더 다양한 요금제 비교 지원

## 프로젝트 목표

클라우드 인프라는 점점 더 복잡해지고 있지만, 서비스를 만드는 문턱은 점점 낮아지고 있습니다.

AI가 코딩을 대신해주는 시대에, **클라우드 비용 관리까지 AI가 대신해줄 수 있다면** — 아이디어를 가진 모든 사람이 비용 걱정 없이 자신의 서비스를 세상에 내놓을 수 있습니다.

CostCutter는 그 간극을 메우는 플랫폼입니다.
