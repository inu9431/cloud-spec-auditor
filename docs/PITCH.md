# CostCutter — AI가 클라우드 비용 전문가를 대신하는 시대

> **"코드를 만드는 건 AI가 해결했다. 이제 남은 문제는 — 그 코드를 세상에 내놓는 비용이다."**

**현재 상태:** 프로토타입 구현 완료 — Django 백엔드 + ELT 데이터 파이프라인 + Next.js 대시보드 + Prefect 오케스트레이션 동작 중. AWS 실계정 연동 테스트 완료.

---

## 이 프로젝트가 시작된 이유

저는 코딩 부트캠프를 수료한 뒤 같은 부트캠프에서 조교로 일하기 시작했습니다.

수강생 질문에 일일이 답변하는 업무가 반복되자, 자동화를 떠올렸습니다. 디스코드 봇을 만들어 **수강생 질문 → LLM 답변 → Django 검증 → 노션 업로드**까지 자동화했습니다.

그리고 이 봇을 AWS에 배포했을 때, **예상하지 못한 비용이 발생했습니다.**

프리티어를 쓰고 있었지만 EC2와 RDS 조합만으로도 비용이 생겼고, 어떤 인스턴스가 과스펙인지, 다른 클라우드가 더 싼지 판단할 기준이 없었습니다.

**인프라를 아는 개발자인 저도 이랬습니다.** 그렇다면 바이브코딩으로 처음 서비스를 배포하는 사람은 어떨까요?

---

## 문제: 만드는 건 쉬워졌지만, 운영하는 건 여전히 어렵다

AI가 코딩을 대신하는 시대가 왔습니다. Cursor, Claude Code, Replit Agent 같은 도구 덕분에 이제 기획만 잘하면 누구나 앱을 만들 수 있습니다. 비개발자도 아이디어를 프로토타입으로 바꿀 수 있는 시대입니다.

**하지만 만든 앱을 세상에 내놓으려면 클라우드에 배포해야 합니다.** 그리고 그 순간부터 전혀 다른 문제가 시작됩니다.

- AWS, GCP, Azure — 어디가 더 싸지?
- 지금 쓰는 인스턴스가 과스펙인 건 아닌지?
- 각 클라우드마다 다른 도구, 다른 가격 체계, 다른 용어

### 이건 작은 문제가 아닙니다

2025년 전 세계 퍼블릭 클라우드 지출은 약 7,230억 달러에 달하며, 이 중 약 21~32%가 과잉 프로비저닝이나 유휴 리소스로 인해 낭비되고 있습니다. 금액으로 환산하면 **연간 약 445억~2,000억 달러 규모**의 낭비입니다.

대기업에는 이 문제를 해결할 FinOps 팀이나 인프라 전문가가 있습니다. 하지만 1인 개발자, 초기 스타트업, 바이브코딩으로 처음 배포하는 메이커에게는 그런 자원이 없습니다. 인프라 전문가 한 명의 연봉은 최소 6,000만~1억 원 수준이며, 10인 이하 스타트업이 감당할 수 있는 비용이 아닙니다.

**결국 인프라 전문가가 없는 팀은 과스펙 인스턴스를 모른 채 비용을 낭비하거나, 감으로 인프라를 선택하고 있습니다.**

---

## 해결: AI가 클라우드 인프라 전문가 역할을 한다

CostCutter는 클라우드 비용 관리에 전문 지식이 없는 사람도 인프라를 최적화할 수 있게 해주는 AI 플랫폼입니다.

### 사용자가 하는 일

AWS Key를 한 번 등록합니다. **끝입니다.**

### AI가 하는 일

1. 현재 인프라를 자동으로 수집합니다
2. AWS, GCP, Azure 3사를 동일한 기준으로 비교합니다
3. 과스펙 인스턴스를 탐지합니다
4. 비용 절감 방법을 **사람이 이해할 수 있는 언어로** 설명합니다

### 예시 결과

```
현재 인프라:  AWS t3.medium  (월 $84)
  - 평균 CPU 사용률: 8%
  - 메모리 사용률: 22%

AI 추천:
  "이 인스턴스는 CPU와 메모리를 20% 미만으로 사용하고 있어 과스펙입니다.
   t3.small로 변경하면 월 $42를 절감할 수 있습니다.
   GCP e2-small을 사용하면 동일 스펙에서 월 $38로 더 절감됩니다."

예상 절감: 연간 약 $504
```

## 왜 기존 서비스와 다른가

| 구분 | 기존 방식 | CostCutter |
|---|---|---|
| **타겟 사용자** | DevOps 엔지니어, 인프라 전문가 | 1인 개발자, 초기 스타트업, 비개발자 메이커 |
| **인프라 전문가 비용** | 연 6,000만~1억 원 채용 필요 | AI가 대체 — 추가 인건비 없음 |
| **클라우드 범위** | 단일 클라우드 내 분석 | AWS / GCP / Azure 3사 통합 비교 |
| **사용 방식** | 대시보드에서 직접 분석 | Key 등록 한 번이면 자동 수집 + 자동 분석 |
| **결과 형태** | 숫자와 차트 | AI가 왜 이 추천을 했는지 자연어로 설명 |
| **비교 기준** | 각 클라우드 자체 단위 사용 | 3사 리전을 동일 기준으로 정규화 후 비교 |

### 핵심 차별점: 3사 리전 정규화

AWS는 `ap-northeast-2`, GCP는 `asia-northeast3`, Azure는 `koreacentral`이라고 부릅니다. 같은 서울 리전인데 이름이 전부 다릅니다.

CostCutter는 이런 리전명을 하나의 정규화된 기준(`NormalizedRegion.KR`)으로 통합합니다. 이를 통해 **사과 대 사과 비교**가 가능해집니다.

## 어떻게 동작하는가

### 전체 흐름

```
유저: AWS Key 등록 (1회)
         ↓
자동 수집: 24시간 주기로 EC2 인스턴스 + 비용 + 사용률 데이터 수집
         ↓
3사 비교: AWS / GCP / Azure 동일 스펙 가격 자동 비교 (주 1회)
         ↓
AI 분석: Python이 비용 계산 → AI가 자연어로 설명 생성
         ↓
대시보드: 유저가 결과 확인
```

### AI의 신뢰성을 어떻게 확보했는가

AI(LLM)에게 숫자 계산을 맡기면 부정확한 결과가 나올 수 있습니다. CostCutter는 이를 구조적으로 방지했습니다.

**Python이 모든 숫자를 계산합니다** — 절감액, 사용률, 스코어링. 결정적(deterministic) 로직이므로 항상 같은 결과가 나옵니다.

**AI는 설명만 생성합니다** — "왜 이 인스턴스가 과스펙인지", "전환 시 고려할 점"을 사람이 이해하기 쉬운 언어로 풀어줍니다.

이 분리 구조 덕분에 AI의 환각(hallucination)이 비용 계산에 영향을 주지 않습니다.

### 보안

CostCutter는 **읽기 전용 권한만 사용**합니다. 유저의 인스턴스를 중지하거나 삭제하는 기능이 없습니다. AWS Key는 암호화되어 DB에 저장되며, API 호출 시점에만 복호화됩니다.

## 우리가 아는 한계

솔직하게 이야기하면, CostCutter에는 현재 다음과 같은 한계가 있습니다.

**EC2 인스턴스에 집중되어 있습니다.** 클라우드 비용의 큰 부분을 차지하는 스토리지(S3), 데이터베이스(RDS), 네트워크 비용은 아직 분석 대상이 아닙니다. EC2 과스펙 감지 → 다른 서비스로 확장하는 로드맵을 가지고 있습니다.

**사용률 분석에 14일이 필요합니다.** AWS Compute Optimizer가 정밀한 과스펙 판단을 위해 14일치 데이터를 필요로 합니다. 처음 가입한 유저에게는 스펙 기반 가격 비교(신뢰도: 중간)를 먼저 제공하고, 14일 후 사용률 기반 정밀 분석(신뢰도: 높음)으로 업그레이드하는 2단계 접근을 설계했습니다.

**On-Demand 가격 기준입니다.** Reserved Instance나 Spot Instance 가격은 아직 포함하지 않습니다. 이미 데이터 모델에 가격 유형 필드가 준비되어 있어, 데이터만 추가하면 확장 가능합니다.

**이 한계들을 멘토링을 통해 보완하고 싶습니다.** 특히 EC2 외 서비스 확장과 가격 모델 다양화는 실제 유저 피드백과 클라우드 도메인 전문가의 조언이 큰 도움이 될 것입니다.

## 실행력 증명

이 프로젝트는 아이디어 단계가 아닙니다. 이미 동작하는 프로토타입이 있으며, 설계 과정에서 마주친 문제들을 체계적으로 해결해왔습니다.

- **아키텍처 의사결정 기록 (ADR) 11건** — 모든 주요 기술 선택에 "왜 이렇게 했는가"를 문서화
- **트러블슈팅 기록 12건** — AWS 실계정 연동 중 발견한 문제와 해결 과정을 기록
- **AWS 실계정 테스트 완료** — 더미 데이터가 아닌 실제 AWS 인프라에서 검증
- **디스코드 봇 → CostCutter** — 이전 프로젝트의 경험(LLM 비용 최적화, 캐시 전략)이 이 프로젝트에 직접 반영

## 비전: 아이디어를 가진 모든 사람이 비용 걱정 없이 세상에 내놓을 수 있는 세상

클라우드 인프라는 점점 더 복잡해지고 있지만, 서비스를 만드는 문턱은 점점 낮아지고 있습니다.

AI가 코딩을 대신해주면서 **"아이디어만 있으면 누구나 만들 수 있는 시대"**가 왔습니다. 하지만 만든 서비스를 세상에 내놓으려면 클라우드 비용이라는 장벽이 남아 있습니다. 이 장벽 때문에 좋은 아이디어가 배포 단계에서 포기되거나, 불필요한 비용을 내면서 운영됩니다.

CostCutter가 꿈꾸는 미래는 이렇습니다: **클라우드에 대해 아무것도 몰라도, AI에게 물어보면 최적의 인프라를 알려주는 세상.** 마치 AI가 코드를 대신 짜주듯이, AI가 인프라 의사결정을 대신 해주는 세상.

이것이 실현되면 개발도상국의 개발자, 비개발자 메이커, 학생 창업가 — 인프라 전문가를 고용할 수 없는 모든 사람이 **자신의 아이디어를 비용 걱정 없이 세상에 내놓을 수 있습니다.**
